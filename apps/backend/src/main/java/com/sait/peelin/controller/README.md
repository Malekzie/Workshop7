V1 of the api, where all the endpoints are defined. The controllers are responsible for handling the incoming requests and returning the appropriate responses. Each controller is annotated with @RestController and @RequestMapping to define the base path for the endpoints. The controllers also use @Autowired to inject the necessary services to handle the business logic.
If breaking changes are to be added, we will add a V2 of the api, and the existing V1 will be maintained for backward compatibility. This allows clients to continue using the existing endpoints while we introduce new features or changes in the V2.
